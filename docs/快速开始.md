# 快速开始指南

> 5分钟内完成第一次使用

## 前置要求

- Git
- Bash shell
- Claude Code 或 Claude.ai 访问权限

## 步骤1：获取项目

```bash
# 安装到本机 Claude skills 目录（推荐）
git clone <your_repo_url> ~/.claude/skills/harness_skill
cd ~/.claude/skills/harness_skill

# 或者下载ZIP并解压
```

## 步骤2：快速初始化

### 方式A：一键快速开始（推荐）

```bash
bash scripts/quickstart.sh
```

这个脚本会自动（在 **skill 目录** 里执行）：
1. 检查环境依赖
2. 创建配置文件
3. 初始化项目结构
4. 运行验证检查
5. 显示下一步建议

> 如果你的目标是在“业务项目目录”里完成项目初始化，请在业务项目目录执行：
>
> ```bash
> cd /path/to/your/project
> bash ~/.claude/skills/harness_skill/scripts/init_project.sh
> ```

### 方式B：手动初始化

```bash
# 1. 复制配置模板
cp config/env.example.json config/env.json

# 2. 编辑配置（可选）
vim config/env.json

# 3. 运行初始化
bash scripts/init_project.sh

# 4. 验证安装
bash scripts/check_setup.sh
```

## 步骤3：第一次使用

### 在Claude中开始

打开Claude，说：

```
我迷路了
```

Claude会自动显示：
- 当前项目状态
- 下一步建议
- 可用的技能命令

### 或使用命令行

```bash
# 查看智能引导
bash scripts/guide.sh

# 查看项目状态
bash scripts/update_state.sh show
```

## 步骤4：开始开发

### 场景1：开发新功能

```
1. 在Claude中说："我想开发一个新功能"
2. Claude会引导你运行 /brainstorming
3. 回答几个问题后，Claude会帮你：
   - 创建设计文档
   - 生成实施计划
   - 创建任务列表
```

### 场景2：修复Bug

```
1. 在Claude中说："我遇到了一个bug"
2. Claude会引导你运行 /systematic-debugging
3. 按照步骤：
   - 理解问题
   - 复现bug
   - 定位原因
   - 实施修复
```

### 场景3：代码审查

```
1. 完成代码后，在Claude中说："帮我审查代码"
2. Claude会运行 /requesting-code-review
3. 获得详细的审查报告和改进建议
```

## 常用命令速查

### 智能引导

```bash
# 查看完整引导
bash scripts/guide.sh

# 查看下一步建议
bash scripts/guide.sh auto

# 查看项目状态
bash scripts/update_state.sh show
```

### 项目管理

```bash
# 检查设置
bash scripts/check_setup.sh

# 更新文档
bash scripts/update_docs.sh

# 创建GitHub issues
bash scripts/create_issues.sh
```

### 在Claude中使用自然语言

不需要记住命令，直接说：

- "我迷路了" → 显示完整引导
- "下一步做什么" → 显示建议
- "查看状态" → 显示项目状态
- "检查设置" → 运行设置检查
- "有什么技能" → 显示技能列表

## 配置说明

### 基本配置（config/env.json）

```json
{
  "PROJECT_NAME": "my-project",
  "PROJECT_DESCRIPTION": "项目描述",
  "GITHUB_USER": "your-username",
  "GITHUB_REPO": "your-repo",
  "GITHUB_TOKEN": "",
  "DATABASE_URL": "mongodb://localhost:27017/mydb",
  "MODEL": "claude-sonnet-3.5",
  "DEFAULT_BRANCH": "main"
}
```

### 必填项

- `PROJECT_NAME` - 项目名称

### 可选项

- `GITHUB_TOKEN` - 只有需要创建GitHub issues时才需要
- `DATABASE_URL` - 只有项目使用数据库时才需要
