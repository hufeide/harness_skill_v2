## 安装和配置问题

### Q1: 运行 check_setup.sh 失败

**症状**：
```bash
bash scripts/check_setup.sh
❌ Claude Code CLI 未安装
```

**解决方案**：

1. 检查 Claude Code 是否安装：
```bash
which claude
```

2. 如果未安装，访问 [Claude Code 官网](https://claude.ai/code) 安装

3. 确保 Claude Code 在 PATH 中：
```bash
export PATH="$PATH:/path/to/claude"
```

---

### Q2: 配置文件不存在

**症状**：
```bash
❌ 配置文件不存在: config/env.json
```

**解决方案**：

```bash
# 复制配置模板
cp config/env.example.json config/env.json

# 编辑配置
vim config/env.json
```

---

### Q3: 环境变量未设置

**症状**：
```bash
❌ 环境变量未设置: PROJECT_NAME
```

**解决方案**：

编辑 `config/env.json`，确保必填字段已设置：

```json
{
  "PROJECT_NAME": "your-project-name",
  "PROJECT_DESCRIPTION": "项目描述"
}
```

---

### Q4: GitHub Token 无效

**症状**：
```bash
❌ GitHub API 请求失败: 401 Unauthorized
```

**解决方案**：

1. 访问 [GitHub Tokens](https://github.com/settings/tokens)
2. 创建新的 Personal Access Token
3. 选择权限：`repo`, `workflow`, `write:packages`
4. 复制 token 到 `config/env.json`：

```json
{
  "GITHUB_TOKEN": "ghp_your_token_here"
}
```

## 脚本执行问题

### Q5: 脚本没有执行权限

**症状**：
```bash
bash: ./scripts/guide.sh: Permission denied
```

**解决方案**：

```bash
# 添加执行权限
chmod +x scripts/*.sh

# 或单个脚本
chmod +x scripts/guide.sh
```

---

### Q6: 脚本找不到命令

**症状**：
```bash
./scripts/guide.sh: line 10: jq: command not found
```

**解决方案**：

安装缺失的依赖：

```bash
# macOS
brew install jq

# Ubuntu/Debian
sudo apt-get install jq

# CentOS/RHEL
sudo yum install jq
```

---

### Q7: 脚本执行卡住

**症状**：
脚本运行后没有任何输出，也不退出

**解决方案**：

1. 按 `Ctrl+C` 终止脚本
2. 检查脚本日志：
```bash
bash -x scripts/guide.sh
```

3. 查看是否有后台进程：
```bash
ps aux | grep guide.sh
kill <pid>
```

### Q10: MCP 服务器无法连接

**症状**：
```
❌ MCP 服务器连接失败: mongodb-mcp-server
```

**解决方案**：

1. 检查 MCP 服务器是否运行：
```bash
ps aux | grep mcp
```

2. 重启 MCP 服务器：
```bash
# 停止
pkill -f mongodb-mcp-server

# 启动
mongodb-mcp-server &
```

3. 检查配置：
```bash
cat ~/.claude/mcp.json
```

## Git 相关问题

### Q12: Git worktree 创建失败

**症状**：
```bash
fatal: 'feature/new-feature' is already checked out at '/path/to/worktree'
```

**解决方案**：

1. 查看现有 worktrees：
```bash
git worktree list
```

2. 删除旧的 worktree：
```bash
git worktree remove /path/to/worktree
```

3. 清理：
```bash
git worktree prune
```

---

### Q13: Git 推送失败

**症状**：
```bash
! [rejected] main -> main (fetch first)
```

**解决方案**：

1. 拉取最新代码：
```bash
git pull origin main --rebase
```

2. 解决冲突（如果有）

3. 重新推送：
```bash
git push origin main
```

---

### Q14: Git 分支冲突

**症状**：
```bash
CONFLICT (content): Merge conflict in file.js
```

**解决方案**：

1. 查看冲突文件：
```bash
git status
```

2. 编辑冲突文件，解决冲突标记：
```
<<<<<<< HEAD
你的修改
=======
别人的修改
>>>>>>> branch-name
```

3. 标记为已解决：
```bash
git add file.js
git commit
```

---

### Q15: 远程仓库未设置

**症状**：
```bash
fatal: No configured push destination
```

**解决方案**：

```bash
# 添加远程仓库
git remote add origin https://github.com/username/repo.git

# 或使用脚本
bash scripts/set_remote.sh https://github.com/username/repo.git

# 推送
git push -u origin main
```

## Hooks 问题

### Q16: Hooks 不执行

**症状**：
提交代码时，pre_commit hook 没有运行

**解决方案**：

1. 检查 hooks 是否存在：
```bash
ls -la ~/.claude/hooks/
```

2. 检查执行权限：
```bash
chmod +x ~/.claude/hooks/*
```

3. 手动测试 hook：
```bash
~/.claude/hooks/pre_commit
```

4. 重新配置 hooks：
```bash
bash scripts/setup_hooks.sh
```

---

### Q17: Hook 执行失败

**症状**：
```bash
pre_commit hook failed with exit code 1
```

**解决方案**：

1. 查看 hook 输出：
```bash
~/.claude/hooks/pre_commit
```

2. 检查 hook 脚本：
```bash
cat ~/.claude/hooks/pre_commit
```

3. 修复错误或临时跳过：
```bash
git commit --no-verify
```

## 性能问题

### Q18: 脚本运行很慢

**症状**：
`check_setup.sh` 运行超过 1 分钟

**解决方案**：

1. 检查网络连接（某些检查需要网络）
2. 禁用不必要的检查
3. 使用缓存：
```bash
export SKIP_NETWORK_CHECKS=1
bash scripts/check_setup.sh
```

---

### Q19: Claude 响应慢

**症状**：
Claude 执行技能命令很慢

**解决方案**：

1. 检查网络延迟：
```bash
ping claude.ai
```

2. 切换到更快的模型：
```json
{
  "MODEL": "claude-sonnet-3.5"
}
```

3. 减少上下文大小

## 其他问题

### Q23: 找不到项目状态

**症状**：
```bash
bash scripts/guide.sh
❌ 状态文件不存在
```

**解决方案**：

```bash
# 创建状态文件
echo "stage=initiation" > .guide_state
echo "last_activity=项目初始化" >> .guide_state
echo "timestamp=$(date '+%Y-%m-%d %H:%M:%S')" >> .guide_state
```

---

### Q24: 迷路了，不知道做什么

**症状**：
不确定下一步该做什么

**解决方案**：

1. 在 Claude 中说：
```
我迷路了
```

2. 或运行：
```bash
bash scripts/guide.sh
```

3. 查看项目状态：
```bash
bash scripts/update_state.sh show
```

---

### Q25: 需要重置项目

**症状**：
项目状态混乱，想重新开始

**解决方案**：

```bash
# 备份重要文件
cp -r src src.backup
cp -r docs docs.backup

# 重新初始化
rm -rf .guide_state .guide_history
bash scripts/init_project.sh

# 恢复代码
cp -r src.backup/* src/
```
