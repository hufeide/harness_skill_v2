## 开发工作流

### 标准开发流程

```
1. 需求分析 (/brainstorming)
   ↓
2. 创建计划 (/writing-plans)
   ↓
3. 创建分支 (/using-git-worktrees)
   ↓
4. TDD 开发 (/test-driven-development)
   ↓
5. 代码审查 (/requesting-code-review)
   ↓
6. 完成验证 (/verification-before-completion)
   ↓
7. 创建 PR (/finishing-a-development-branch)
```

### 快速迭代流程

适用于小功能或 bug 修复：

```
1. 明确问题
   ↓
2. 编写测试
   ↓
3. 实现功能
   ↓
4. 运行测试
   ↓
5. 提交代码
```

### 并行开发流程

使用 Git worktrees 并行开发多个功能：

```bash
# 功能 A
cd ~/project-feature-a
/brainstorming
# 开发...

# 功能 B（新窗口）
cd ~/project-feature-b
/brainstorming
# 开发...
```

**优势**：
- 快速切换上下文
- 独立的工作环境
- 共享 Git 历史

## 代码提交规范

### Commit Message 格式

使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(auth): 添加用户登录功能` |
| `fix` | Bug 修复 | `fix(api): 修复用户查询接口错误` |
| `docs` | 文档更新 | `docs(readme): 更新安装说明` |
| `style` | 代码格式 | `style(lint): 修复代码格式问题` |
| `refactor` | 重构 | `refactor(user): 重构用户模型` |
| `test` | 测试 | `test(auth): 添加登录测试用例` |
| `chore` | 构建/工具 | `chore(deps): 更新依赖版本` |
| `perf` | 性能优化 | `perf(query): 优化数据库查询` |

### 好的 Commit 示例

```
feat(auth): 实现 JWT 认证功能

- 添加 JWT token 生成和验证
- 实现登录/登出 API
- 添加认证中间件保护路由
- 添加单元测试和集成测试

Closes #123
```

### 不好的 Commit 示例

❌ `fix bug`
❌ `更新代码`
❌ `WIP`
❌ `修改了一些东西`

### 提交频率

- ✅ 小步提交，频繁提交
- ✅ 每个提交只做一件事
- ✅ 提交前确保代码可运行
- ❌ 不要积累大量修改后一次提交

## 分支管理策略

### 分支命名规范

```
<type>/<description>

示例：
feature/user-authentication
bugfix/login-error
hotfix/security-patch
refactor/database-layer
```

### 分支类型

| 分支类型 | 说明 | 生命周期 |
|---------|------|---------|
| `main` | 主分支，随时可部署 | 永久 |
| `develop` | 开发分支 | 永久 |
| `feature/*` | 功能分支 | 临时 |
| `bugfix/*` | Bug 修复分支 | 临时 |
| `hotfix/*` | 紧急修复分支 | 临时 |
| `release/*` | 发布分支 | 临时 |

### 分支工作流

#### Git Flow（推荐用于大型项目）

```
main (生产)
  ↑
release/v1.0
  ↑
develop (开发)
  ↑
feature/new-feature
```

#### GitHub Flow（推荐用于小型项目）

```
main
  ↑
feature/new-feature
```

### 分支保护规则

对 `main` 和 `develop` 分支：

- ✅ 要求 PR 审查
- ✅ 要求 CI 通过
- ✅ 禁止强制推送
- ✅ 要求分支最新

## 测试策略

### 测试金字塔

```
       /\
      /E2E\      10% - 端到端测试
     /------\
    /集成测试\    20% - 集成测试
   /----------\
  /  单元测试  \  70% - 单元测试
 /--------------\
```

### 测试覆盖率目标

- **总体覆盖率**: > 80%
- **核心业务逻辑**: > 90%
- **工具函数**: > 95%

### 必须测试的场景

1. **用户认证和授权**
   ```javascript
   describe('Authentication', () => {
     it('should login with valid credentials', async () => {
       // 测试代码
     });
     
     it('should reject invalid credentials', async () => {
       // 测试代码
     });
   });
   ```

2. **数据 CRUD 操作**
   ```javascript
   describe('User CRUD', () => {
     it('should create a new user', async () => {});
     it('should read user data', async () => {});
     it('should update user data', async () => {});
     it('should delete user', async () => {});
   });
   ```

3. **边界条件**
   ```javascript
   it('should handle empty input', () => {});
   it('should handle null values', () => {});
   it('should handle large datasets', () => {});
   ```

4. **错误处理**
   ```javascript
   it('should throw error for invalid input', () => {
     expect(() => func(null)).toThrow();
   });
   ```

### TDD 工作流

```
1. 编写失败的测试（Red）
   ↓
2. 编写最少代码让测试通过（Green）
   ↓
3. 重构代码（Refactor）
   ↓
4. 重复
```

### 测试命名规范

```javascript
// 好的测试名称
it('should return user when valid ID is provided', () => {});
it('should throw error when user not found', () => {});

// 不好的测试名称
it('test1', () => {});
it('works', () => {});
```

### 审查流程

```
1. 自我审查
   ↓
2. 运行 /requesting-code-review
   ↓
3. 根据语言运行专业审查
   - TypeScript: /compound-engineering:review:kieran-typescript-reviewer

   - Rails: /compound-engineering:review:kieran-rails-reviewer
   ↓
4. 修复问题
   ↓
5. 请求人工审查
```

### 审查反馈原则

**给出反馈时**：
- ✅ 具体指出问题
- ✅ 提供改进建议
- ✅ 解释原因
- ❌ 不要只说"不好"

**接收反馈时**：
- ✅ 保持开放心态
- ✅ 理解审查意图
- ✅ 讨论不同观点
- ❌ 不要防御性回应

## 性能优化

### 数据库优化

#### 1. 索引优化
```sql
-- 为常用查询字段添加索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_posts_user_id ON posts(user_id);
```

#### 2. 避免 N+1 查询
```javascript
// ❌ N+1 查询
const users = await User.findAll();
for (const user of users) {
  const posts = await user.getPosts(); // N 次查询
}

// ✅ 使用 include
const users = await User.findAll({
  include: [Post] // 1 次查询
});
```

#### 3. 分页查询
```javascript
// ✅ 使用分页
const users = await User.findAll({
  limit: 20,
  offset: page * 20
});
```

### 缓存策略

#### 1. 应用层缓存
```javascript
const cache = new Map();

async function getUser(id) {
  if (cache.has(id)) {
    return cache.get(id);
  }
  
  const user = await User.findById(id);
  cache.set(id, user);
  return user;
}
```

#### 2. Redis 缓存
```javascript
const cached = await redis.get(`user:${id}`);
if (cached) {
  return JSON.parse(cached);
}

const user = await User.findById(id);
await redis.set(`user:${id}`, JSON.stringify(user), 'EX', 3600);
```

## 安全实践

### 输入验证

```javascript
// ✅ 验证所有输入
function createUser(data) {
  const schema = Joi.object({
    email: Joi.string().email().required(),
    password: Joi.string().min(8).required(),
    age: Joi.number().min(0).max(150)
  });
  
  const { error, value } = schema.validate(data);
  if (error) throw new Error('Invalid input');
  
  return User.create(value);
}
```

### SQL 注入防护

```javascript
// ❌ 不安全
const query = `SELECT * FROM users WHERE email = '${email}'`;

// ✅ 使用参数化查询
const query = 'SELECT * FROM users WHERE email = ?';
const users = await db.query(query, [email]);
```

### XSS 防护

```javascript
// ✅ 转义用户输入
import escape from 'escape-html';

const safeContent = escape(userInput);
```

### 敏感信息保护

```javascript
// ✅ 不要在代码中硬编码密钥
const apiKey = process.env.API_KEY;

// ✅ 不要记录敏感信息
logger.info('User logged in', { 
  userId: user.id 
  // ❌ password: user.password 
});
```

### 认证和授权

```javascript
// ✅ 使用成熟的认证库
import jwt from 'jsonwebtoken';

// ✅ 实现权限检查
function requireAdmin(req, res, next) {
  if (!req.user.isAdmin) {
    return res.status(403).json({ error: 'Forbidden' });
  }
  next();
}
```
