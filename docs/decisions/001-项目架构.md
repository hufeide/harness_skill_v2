# ADR-001: 项目架构设计

## 状态

已接受

## 日期

2026-03-09

## 上下文

需要为 Claude 技能库设计一个清晰、可维护的项目架构，以支持：

1. 智能引导系统
2. 自动化工作流
3. 多语言支持
4. 插件扩展
5. 文档管理

## 决策

采用模块化架构，分为以下几个核心模块：

### 1. 核心模块

```
harness_skill/
├── config/          # 配置管理
├── scripts/         # 自动化脚本
├── docs/            # 文档系统
├── examples/        # 示例项目
└── templates/       # 项目模板
```

### 2. 配置系统

使用 JSON 格式配置文件：
- `config/env.json` - 环境变量
- `config/project.json` - 项目配置
- `config/permissions.json` - 权限配置
- `skill.json` - 技能配置

**理由**：
- JSON 易于解析和编辑
- 支持版本控制
- 跨平台兼容

### 3. 脚本系统

使用 Bash 脚本实现自动化：
- 跨平台兼容（Linux/macOS）
- 无需额外依赖
- 易于调试和维护

**核心脚本**：
- `quickstart.sh` - 快速开始
- `init_project.sh` - 项目初始化
- `guide.sh` - 智能引导
- `check_setup.sh` - 设置验证

### 4. 文档系统

分层文档架构：

```
docs/
├── 快速开始.md      # 新手指南
├── 完整指南.md      # 详细文档
├── API参考.md       # 命令参考
├── 最佳实践.md      # 工作流技巧
├── 故障排除.md      # FAQ
├── plans/           # 设计文档
└── decisions/       # ADR 记录
```

**理由**：
- 清晰的信息层次
- 易于查找和维护
- 支持渐进式学习

### 5. 状态管理

使用文件系统存储状态：
- `.guide_state` - 当前状态
- `.guide_history` - 活动历史

**格式**：
```
stage=development
last_activity=实现用户认证功能
timestamp=2026-03-09 10:30:00
```

**理由**：
- 简单可靠
- 易于读写
- 支持版本控制

### 6. 插件系统

支持三类插件：
1. **Skills** - Claude 技能
2. **MCP** - Model Context Protocol 服务器
3. **Compound Engineering** - 高级工作流

**配置方式**：
```json
{
  "plugins": {
    "skills": ["brainstorming", "writing-plans"],
    "mcp": ["mongodb-mcp-server"],
    "compound-engineering": "@EveryInc/..."
  }
}
```

## 后果

### 优点

1. **模块化**：各模块职责清晰，易于维护
2. **可扩展**：支持插件和模板扩展
3. **易用性**：分层文档，渐进式学习
4. **自动化**：脚本系统支持自动化工作流
5. **跨平台**：Bash 脚本跨平台兼容

### 缺点

1. **Bash 限制**：Windows 支持需要 WSL 或 Git Bash
2. **文件系统依赖**：状态管理依赖文件系统
3. **学习曲线**：概念较多，需要时间学习

### 缓解措施

1. **Windows 支持**：
   - 提供 PowerShell 版本脚本
   - 文档说明 WSL 安装

2. **状态管理**：
   - 考虑未来支持数据库存储
   - 提供状态导入导出功能

3. **学习曲线**：
   - 提供快速开始指南
   - 提供示例项目
   - 提供交互式教程

## 替代方案

### 方案 A：Python 实现

**优点**：
- 跨平台更好
- 更强大的功能
- 更好的错误处理

**缺点**：
- 需要 Python 环境
- 增加依赖复杂度
- 部署更复杂

**为什么不选**：增加了使用门槛

### 方案 B：Node.js 实现

**优点**：
- JavaScript 生态丰富
- 前端开发者熟悉
- npm 包管理方便

**缺点**：
- 需要 Node.js 环境
- 启动速度较慢
- 内存占用较大

**为什么不选**：不是所有用户都有 Node.js

### 方案 C：Go 实现

**优点**：
- 编译为单一二进制
- 性能优秀
- 跨平台编译

**缺点**：
- 需要编译步骤
- 修改不够灵活
- 学习成本高

**为什么不选**：失去了脚本的灵活性
